Introduction

Welcome to Renarration, an innovative platform designed to re-narrate existing web content to match users' literacy and understanding levels. Our app utilizes a unique concept called SWeeTs and SWeeT Web to transform how web content is consumed and understood. This documentation will guide you through the functionalities of Renarration, including the creation of SWeeTs and Renarrations.

Who

Janastu is an organization working for local communities to make the web more inclusive. Founded by a group of computer scientists in 1999, Janastu focuses on leveraging technology to empower local communities.

What

Renarration is a tool designed to create locally-owned web content. It enables users to re-narrate existing web content to better suit the literacy and understanding levels of different audiences.

Where

The Renarration platform can be accessed atsweets.janastu.org.

When

Janastu began its journey in 1999 when a group of computer scientists came together to work towards making the web more inclusive for local communities.

Why

The goal of Renarration is to include local communities in the digital space, making the web more accessible and understandable to everyone, regardless of their technical literacy.

How

Renarration works by allowing technically literate individuals to assist those with lower technical literacy. Through this collaborative effort, they can re-narrate web content, making it more inclusive and easier to understand.

SWeeTs Section

What is SWeeT?

The Social Web of today is characterized by participatory content creation and syndicated communication. Wikipedia represents participatory content creation, while micro-blogging platforms like Twitter facilitate communication. Alongside, the Semantic Web has steadily infused web data with semantics. The Social Semantic Web emerged from these efforts, blending the social and semantic aspects of the web.

A SWeeT, short for "semantic web tweet," is an elementary unit of structured information that allows people to pronounce a semantic relationship of information on the web. SWeeTs are decentralized and structured to reflect semantic relationships between web elements governed by an ontology. They serve as the foundation for Renarration and the interconnecting archives work.

Creating a SWeeT

Step 1: Input URL

  1. Open the Renarration Website: Navigate to the Renarration website on your browser.
  1. Locate the Input Bar: At the top of the homepage in the SWeeTs section, you will find an input bar with a placeholder text prompting you to enter a URL.
  1. Paste the URL: Copy the URL of the web page you wish to re-narrate and paste it into the input bar.
  1. Create a SWeeT: Click the "Create SWeeT" button next to the input bar to start the annotation process.

Step 2: Annotation Page

After clicking the "Create SWeeT" button, you will be navigated to the annotation page where you can annotate the web page.

Annotation Page Walkthrough

  • Annotation Mode Switch: Located in the navbar, this switch allows you to toggle annotation mode on or off.
  • Annotation Count and Reference: Displays the number of existing annotations on the page and provides references to them.
  • URL Input: Similar to the homepage input, this allows you to enter a new URL to annotate.

Step 3: Annotating the Web Page

The annotation body displays the web page of the given URL. Here's how to annotate:

  1. Annotation Mode Off: When annotation mode is off, you can click on links and browse the page normally.
  1. Annotation Mode On: When annotation mode is on, you can select elements on the page to annotate.

To annotate:

  1. Select an Element: Click on the element you wish to annotate.
  1. Open Annotator: A modal, called the annotator, will pop up.
  1. Annotator Features:
  • Selected Element View: Displays the element you selected.
  • Input Box: Provides a space to write the re-narrated content.
  • Upload Media: Allows you to upload related media (up to 5 MB).
  • Tags: Add up to 3 tags to categorize the annotated content.
  • Buttons: "Cancel" to discard changes and "Add Annotation" to save the annotation.

You can add multiple annotations to a page by repeating the above steps. Navigate to new URLs using the input in the navbar or by clicking links on the page when annotation mode is off.

Step 4: Creating a SWeeT

Navigate to the SWeeT creation page to finalize your SWeeT.

  • List of Annotations: Displays all the annotations you made for the URL.
  • SWeeT Title: Provide a title for your SWeeT.
  • Annotations Search Bar: Add existing annotations from the Renarration database to your SWeeT using tags for search optimization.
  • Edit/Delete Annotations: Options to edit or delete annotations.

After finalizing the annotations and title, click the "Create SWeeT" button. A popup will confirm whether you want your SWeeT to have private or public access. Public SWeeTs can be added to other SWeeTs, while private ones cannot. Upon confirmation, your SWeeT is created, and you will receive a sharing ID.

What is Renarration Page?

Renarration is a tool designed to create locally-owned web content by re-narrating existing web content to better suit the literacy and understanding levels of different audiences. It enables users to annotate and customize web pages, making them more accessible and inclusive for various literacy levels and technical proficiencies.

Creating a Renarration page

A Renarration is a collection of SWeeTs for a specific URL. You can group different SWeeTs that share common themes into a single Renarration.

Step 1: Input URL

  1. Open the Renarration Website: Navigate to the Renarration website on your browser.
  1. Locate the Renarration URL Input Bar: Similar to the SWeeT creation input bar, but specifically for Renarrations.
  1. Paste the URL: Copy the URL of the web page you wish to renarrate and paste it into the input bar.
  1. Create Renarration: Click the "Create Renarration" button next to the input bar to start the renarration process.

Step 2: Compose Renarration

You will be navigated to the Renarration compose page, where you can select SWeeTs for your Renarration.

  • List of SWeeTs: Displays all the SWeeTs available for the URL.
  • Add SWeeTs: Choose the SWeeTs you want to add to the Renarration by clicking the "Add SWeeT" button next to each SWeeT.

Step 3: Renarration Creation

After selecting the SWeeTs, you will be taken to the Renarration creation page.

  • Renarration Title: Provide a title for your Renarration.
  • Create Renarration: Click the "Create Renarration" button. A popup will confirm the creation of the Renarration.

Upon confirmation, your Renarration is created, and you will receive a sharing ID, which can be used for editing the Renarration.

Use Cases

Use Case 1: Educational Content Adaptation

Scenario: A teacher wants to adapt a complex article on fundamental rights for their students.

Steps:

  1. Create SWeeTs: The teacher creates individual SWeeTs for each fundamental right by annotating a detailed legal document.
  1. Compose Renarration: The teacher groups these SWeeTs into a single Renarration titled "Fundamental Rights for Students."
  1. Share Renarration: The teacher shares the Renarration with their students, who can now access simplified explanations of each fundamental right.

Use Case 2: Multilingual Website

Scenario: A website wants to provide content in multiple languages to cater to a diverse audience.

Steps:

  1. Create SWeeTs: Content creators annotate web pages in different languages, creating SWeeTs for each language version.
  1. Compose Renarration: The website owner groups these SWeeTs into Renarrations for each language, such as "Website in Spanish" and "Website in French."
  1. Share Renarration: The website owner shares these Renarrations with users, who can now access the website content in their preferred language.

Use Case 3: Accessibility Enhancement

Scenario: A non-profit organization wants to make web content more accessible for people with cognitive disabilities.

Steps:

  1. Create SWeeTs: Volunteers create SWeeTs by simplifying web content and adding multimedia aids.
  1. Compose Renarration: The organization groups these SWeeTs into Renarrations based on topics, such as "Simplified News" or "Easy-to-Read Guides."

3. Share Renarration: The organization shares these Renarrations with their audience, who can now access web content tailored to their needs.

Renarration provides a robust and flexible platform for creating personalized re-narrations of web content. By leveraging the power of SWeeTs and Renarrations, you can enhance the accessibility and understanding of web content for various literacy levels and preferences. Start creating your SWeeTs and Renarrations today and contribute to a more inclusive web!
